# DayZ Ōåö Takaro Integration

A **server-side-only** DayZ mod that connects a modded DayZ server to the [Takaro](https://takaro.io) game-server-management platform. Players do **not** need to download anything to join ŌĆö the mod is loaded with `-serverMod=`, so it lives entirely on the server.

## What it does

**Outbound (DayZ ŌåÆ Takaro):**
- `player-connected` / `player-disconnected`
- `chat-message` (when paired with CF/VPP/Expansion chat hooks)
- `player-death` (with attacker, weapon, position)
- raw RPT log lines (optional)

**Inbound (Takaro ŌåÆ DayZ):**
- `listPlayers`, `getPlayer`, `testReachability`
- `sendMessage` (broadcast or whisper)
- `kickPlayer`, `banPlayer`, `unbanPlayer`
- `teleportPlayer`, `giveItem`
- `executeConsoleCommand` (limited ŌĆö see Architecture)

## Quick start

1. **Install [DayZ Tools](https://store.steampowered.com/app/830640/DayZ_Tools/)** from Steam (free).
2. **Build:** `pwsh scripts/build.ps1` ŌåÆ produces `@TakaroIntegration/Addons/TakaroIntegration.pbo`.
3. **Deploy:** `pwsh scripts/deploy.ps1` ŌåÆ copies `@TakaroIntegration/` to `\\SERVER\GameServers\dayz\`.
4. **Add to server start command** as `-serverMod="@TakaroIntegration"`. **Never use `-mod=`** for this mod ŌĆö that would force every joining client to download it.
5. **Boot the server once** so it writes the default config to `profiles\TakaroIntegration\config.json`.
6. **Edit the config:** set `TakaroApiUrl` and either `RegistrationToken` (one-shot bootstrap) or `IdentityToken` + `GameServerId` (already-registered server).
7. **Restart the server.**

Detailed steps: [docs/INSTALLATION.md](docs/INSTALLATION.md).

## Why a mod and not an external bridge?

The original brief recommended a Node.js bridge tailing the RPT log + BattlEye RCON. We pivoted to an in-game mod because:

- **Lower latency.** Events flow through CF / Mission / PlayerBase hooks instead of being log-scraped after the fact.
- **Higher fidelity.** We can pull `PlayerIdentity`, attacker entity, position, etc. directly from the engine ŌĆö no regex on log lines.
- **One process.** No separate service to keep alive, no port to open, no extra auth surface.
- **Server-only.** `-serverMod=` means clients never download it; players just connect normally.

The trade-off is that DayZ's Enforce Script `RestApi` is HTTP-only (no WebSocket) and has only a basic header API, so we use Takaro's polling/POST adapter shape ŌĆö same model the generic adapter in the Takaro monorepo uses.

## Workshop publishing

See [docs/WORKSHOP_PUBLISHING.md](docs/WORKSHOP_PUBLISHING.md). Short version: open `Publisher Tool` from DayZ Tools, point it at `@TakaroIntegration/`, set the visibility to public, upload. The first publish writes `meta.cpp` with the published Workshop ID.
