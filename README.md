# DayZ тЖФ Takaro Integration

A **server-side-only** DayZ mod that connects a modded DayZ server to the [Takaro](https://takaro.io) game-server-management platform. Players do **not** need to download anything to join тАФ the mod is loaded with `-serverMod=`, so it lives entirely on the server.

## What it does

**Outbound (DayZ тЖТ Takaro):**
- `player-connected` / `player-disconnected`
- `chat-message` (when paired with CF/VPP/Expansion chat hooks)
- `player-death` (with attacker, weapon, position)
- raw RPT log lines (optional)

**Inbound (Takaro тЖТ DayZ):**
- `listPlayers`, `getPlayer`, `testReachability`
- `sendMessage` (broadcast or whisper)
- `kickPlayer`, `banPlayer`, `unbanPlayer`
- `teleportPlayer`, `giveItem`
- `executeConsoleCommand` (limited тАФ see Architecture)

## Quick start

You have two build paths:

**Option A тАФ DayZ Tools (recommended for production / signed builds):**
1. Install [DayZ Tools](https://store.steampowered.com/app/830640/DayZ_Tools/) from Steam (free).
2. `pwsh scripts/build.ps1` тЖТ wraps `AddonBuilder.exe`.

**Option B тАФ Bundled Python packer (no toolchain needed):**
1. `python3 scripts/pack_pbo.py src/takaro_integration @TakaroIntegration/Addons/TakaroIntegration.pbo --prefix TakaroIntegration`
2. The packer writes a valid uncompressed PBO with `product=dayz ugc` and `PboType=Arma Addon` тАФ confirmed loadable on DayZ 1.29.

Then:

3. **Deploy:** `pwsh scripts/deploy.ps1` (copies `@TakaroIntegration/` to `\\SERVER\GameServers\dayz\`), or copy the folder manually.
4. **Add to server start command** as `-serverMod="@TakaroIntegration"`. **Never use `-mod=`** тАФ that would force every joining client to download it.
5. **Boot the server once** so it writes the default config to `profiles\TakaroIntegration\config.json`.
6. **Edit the config:** set `TakaroApiUrl` and either `RegistrationToken` (one-shot bootstrap) or `IdentityToken` + `GameServerId` (already-registered server).
7. **Restart the server.**

### Local end-to-end test

Without a real Takaro account you can verify the full HTTP cycle against a bundled mock:

```bash
python3 scripts/mock_takaro.py --port 8088 &
# In config.json, set TakaroApiUrl=http://localhost:8088 and RegistrationToken=anything
# Restart the DayZ server.
curl -s http://localhost:8088/admin/state | jq    # see what the bridge has POSTed
curl -X POST http://localhost:8088/admin/queue -H 'Content-Type: application/json' \
    -d '{"action":"sendMessage","args":{"message":"hello"}}'
# Within ~2s the bridge polls, dispatches, broadcasts in-game, and POSTs the result back.
```

Detailed steps: [docs/INSTALLATION.md](docs/INSTALLATION.md).

## Why a mod and not an external bridge?

The original brief recommended a Node.js bridge tailing the RPT log + BattlEye RCON. We pivoted to an in-game mod because:

- **Lower latency.** Events flow through CF / Mission / PlayerBase hooks instead of being log-scraped after the fact.
- **Higher fidelity.** We can pull `PlayerIdentity`, attacker entity, position, etc. directly from the engine тАФ no regex on log lines.
- **One process.** No separate service to keep alive, no port to open, no extra auth surface.
- **Server-only.** `-serverMod=` means clients never download it; players just connect normally.

The trade-off is that DayZ's Enforce Script `RestApi` is HTTP-only (no WebSocket) and has only a basic header API, so we use Takaro's polling/POST adapter shape тАФ same model the generic adapter in the Takaro monorepo uses.
